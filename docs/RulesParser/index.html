RulesParser

Transcription d'une grammaire​

On cherche à transcrire une chaîne texte contenant les règles d'une grammaire en des fonctions écrites en C++ qui permettront, après compilation, de tester la validité d'expressions pour cette grammaire. Considérons cet exemple de grammaire :

string test_grammar = R"(
   operande: one of
            a b

   operator: one of
           + *

   exp:
     operande
     operande operator operande
     exp operator ( exp )
     exp operator exp

)";
attention

Chaque règle doit être terminée par un saut de ligne. (ligne vierge)

Voici des expressions valides ou non valides pour cette grammaire :

  • Expressions valides
  • Expressions non valides
  • a
  • b
  • a+b
  • a*b
  • b*(b+a)
  • a+a*b*(b+(b*a))+a
  • c
  • a+c
  • a+b*(b+c)*b
  • a+b*(b+a)*c
  • (a+b)*c
  • a + b

La règle ne précisant pas qu'il faut ignorer les whitespaces, on voit comme dans le dernier exemple qu'une expression qui pourrait paraître valide ne l'est pas si les opérandes ou opérateurs sont entourés d'espaces.

On voit qu'une expression mếme globalement non valide est parsée jusqu'à ce qu'on puisse en extraire une partie valide.

On initialise le parser avec la grammaire et on lance le parsage :

RulesParser parser(test_grammar);
parser.parse();
