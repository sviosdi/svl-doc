Code de RulesParser

Analyse de la grammaire et génération du code​

La classe RulesParser​

En plus des classiques membres ou fonctions nécessaires pour parser : at, ch, next(), isWS(), ws() la classe RulesParser dispose des membres :

std::map<std::string, Rule> rules_names_map;
std::vector<Rule> rules_vect;

Prenons pour exemple la grammaire suivante :

 operande: one of
            a b

   operator: one of
           + *

   exp:
     operande
     operande operator operande
     exp operator ( exp )
     exp operator exp

Une fois la grammaire parsée par :

RulesParser parser(test_grammar);
parser.parse();

Le parser maintiendra en mémoire une organisation décrite par le schéma suivant :

parser.getAllRules() retourne une référence vers le vecteur rules_vect qui contient chacune des règles de la grammaire traduites en un type Rule.

A ce stade du développement les règles sont partagées en deux catégories :

  • Les règles qui vérifient Rule.isOneOf() === true (on les dira de type isOneOf) :
    Elles définissent des 'mots' réservés, des keywords, des caractères particuliers... Une telle règle étant vérifiée quand l'un des 'mots' ou 'caractères' (qu'on appellera des specials) est présent à l'emplacement du caractère courant de parsage.
  • Les règles qui vérifient Rule.isOneOf() === false :
    Elles sont définies par plusieurs lignes de règles. Chaque ligne est de type RuleLine qui contient un vector items de strings où chaque item peut être ou le nom d'une règle déjà définie (et très souvent pour récursion le nom de la règle courante elle même) ou dans le cas contraire ce sera un special : un mot réservé, une parenthèse, un crochet, un caractère particulier...etc. Chaque RuleLine est un élément du vector rules membre de la règle.

Le code généré et son utilisation.​

Transcription des règles en code.​

remarque

Le code indiqué ci-dessous est le code généré par le parser de grammaire. La classe Parser ci-dessous n'intervient pas dans la phase d'analyse des règles et de génération du code (c'est le rôle de RulesParser), mais dans la phase d'utilisation du code généré puis compilé pour tester la validité d'expressions.

Chaque règle est traduite en code par une méthode de la classe Parser dont le nom commence par rl_ suivi du nom de la règle.
Ainsi la règle operande sera traduite par la règle : ParsedNode Parser::rl_operande().
Une expression (de type std::string) sera ainsi globalement testée pour notre grammaire par la fonction Parser::rl_exp() qui retourne un ParsedNode.

std::string toParse{"a+b*a*(b+a)*c"}
Parser parser(toParse);
ParsedNode node = parser.rl_exp();
cout << toparse << ": " << std::boolalpha << node.ok << (node.ok ? ": " + node.value : ": syntax error");
cout << endl;

L'exécution du code précédent donnera :

a+b*a*(b+a)*c: true: a+b*a*(b+a)

Noeud de parsage ParsedNode​

Un noeud de parsage ParsedNode retourné par les fonctions de parsage rl_.... est défini ainsi :

class ParsedNode {
public:
  ParsedNode(ParserData& parser, std::string kind);
  bool ok;
  std::string value;
  size_t begin;
  size_t end;
  std::string kind;
  prs_error error;
  ParserData& parser;
  bool exit;
  std::vector<ParsedNode> children;
  ParsedNode& close(bool ok, bool exit=false);
};
  • ok à true indique que la fonction de parsage a correctement parsé autrement dit qu'elle a pu progresser d'un certain nombre (non nul) de caractères qui respectent la règle qu'elle représente.
    ok à false indique donc que le parsage n'a pas du tout pu progresser.
  • value : la chaîne qui a pu être correctement parsée.
  • begin: la position du caractère courant à l'entrée dans la fonction de parsage.
  • end : la position du caractère courant à la sortie de la fonction de parsage.
        Si value = "Vincent" .... 51 52 [V 53][i 54] .......[t 59] 60 61 ... alors begin = 53 et end = 60.
  • kind : le nom de la règle de grammaire utilisée (i.e. le nom de la règle sans rl_).
  • error: représente l'erreur lorsque ok == false.
  • parser : une référence vers le parser.
  • exit : indique s'il doit y avoir interruption ou non lorsque ok == false.
  • children : les noeuds résultats des lignes de règles qui définissent la fonction de parsage courante.
  • close() est une méthode qui intervient en fin de parsage de la règle. (explications plus loin)

Le parser - classe ParserData​

Le parser dérive de la classe ParserData.

struct ParserData {
  ParserData(std::string s) : unit{s}, at{0}, ch{1} {}
  std::string unit;
  size_t at;
  char ch;
  Ns::Array_stack<size_t> ats_stack;
  char next();
  bool isWS(char c);
  void ws();
  void save();
  void restore();
  void unsave();
  bool isThere(std::string word);
};
  • unit : l'unité à parser
  • at : la position du caractère qui suit le caractère courant
  • ch : le caractère courant (une position avant at) initialisé à une valeur quelconque différente de '\0' (=0)
  • ats_stack : la pile des positions de at à l'entrée dans une fonction de parsage.
  • next() : incrémente at d'une unité et retourne le caractère suivant le caractère courant actuel qui devient le nouveau caractère courant.
    Sans effet si ch est le caractère de fin de chaîne (0)
  • isWS(c) : retourne true si c est un whitespace ' ', '\t', '\n', '\r'.
  • ws() : avance le caractère courant jusqu'au prochain caractère qui n'est pas un whitespace.
    Sans aucun effet si le caractère courant n'est pas lui-même un whitespace ou si on est en fin de chaîne à parser (ch = 0)
  • save() : sauvegarde la position de at sur la pile des positions
  • restore() : récupère la dernière position de at sur la pile et ajuste ch en fonction
  • unsave() : efface la dernière position de at sauvegardée sur la pile sans affecter at
  • isThere(word) : vérifie si le mot|caractère word est présent à la position courante sans modifier ni at ni ch.
    La position courante est inchangée.

Le parser - classe Parser​

struct Parser : public ParserData {
  Parser(std::string unit) : ParserData(unit) { next(); }
  ParsedNode rl_operande();
  ParsedNode rl_operator();
  ParsedNode rl_exp();
  ParsedNode term_(std::string term);
  bool applyRule(ParsedNode& parent, ParsedNode(Parser::*rule)());
  bool applyRule(ParsedNode& parent, std::string term);
};

Le parser contient les trois fonctions de parsage générées par RulesParser.

Principe de parsage​

Chaque règle vérifiée retourne un noeud de parsage avec ok = true

Pour parser la règle exp :

  exp:
     operande
     operande operator operande
     exp operator ( exp )
     exp operator exp

attention

Les lignes (RuleLines 3 et 4, commencent toutes deux par la règle exp qui est la règle en cours de test : on dira qu'elles sont récursives sur la gauche. Dans ce cas, pour ne pas créer une boucle infini, on procèdera ainsi :

  • Si l'une des lignes précédentes n'est pas validée, alors exp ne peut pas être validée est on marquera le noeud retourné avec ok = false.
  • Sinon c'est que exp est vérifiée et on passera sa vérification en commençant la vérification à l'item suivant operator après avoir ajusté ch et at en conséquence.

  • Il y a quatre RuleLines qu'il faut parser dans l'ordre et retenir le dernier noeud valide retourné par chacune d'elle. On prépare pour cela un tableau parents de quatre noeuds initialisés avec kind = "exp" et begin = position du caractère courant et un pointeur good_node qui pointera progressivement vers le dernier noeud valide.

  • Pour chaque ligne :

    • on initialise un tableau correspondant aux items de la ligne où pour chaque item qui est une règle on lui fait correspondre un pointeur vers cette règle et si l'item est un special (i.e. un terminal) on lui fera correspondre nullptr. Ce qui donne par exemple pour la ligne 3 :
      ParseFunction items_3[5]{&Parser::rl_exp, &Parser::rl_operator, nullptr,&Parser::rl_exp, nullptr}

    • on parse la ligne en utilisant les fonctions du tableau précédent qui si elles sont correctement parsées fourniront un noeux avec ok = true qui sera ajouté aux children du parent, ce dernier étant la règle en cours de parsage, autrement dit le noeud parents[i-1] correspondant à la i-ème ligne en cours de parsage.
